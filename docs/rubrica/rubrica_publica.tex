La práctica se valora sobre 10 puntos. La corrección tendrá en cuenta el funcionamiento del programa, la coherencia entre UML e implementación y la defensa realizada.

\section{Techos de nota}
\begin{itemize}
    \item No compila: máximo 3.00.
    \item Uso de estructuras explícitamente prohibidas: máximo 3.00.
    \item No se supera la defensa: máximo 3.00.
\end{itemize}

\section{Penalizaciones globales}
\begin{itemize}
    \item El diagrama UML no se corresponde con el código implementado.
    \item Limpieza general (código comentado, muerto o de pruebas).
    \item Nombres o formato muy deficientes.
    \item Más de un \texttt{return} en una función.
    \item Uso de \texttt{break} fuera de \texttt{switch}.
    \item Uso de \texttt{continue}.
\end{itemize}

\section{Diagrama UML - 3 puntos}
\subsection{Relaciones - 0.5}
\begin{itemize}
    \item Relaciones entre las clases principales.
    \item Tipo de relación correctamente representado.
    \item Multiplicidad cuando corresponda.
\end{itemize}
\subsection{Clases - 0.5}
\begin{itemize}
    \item Clases correctamente identificadas.
    \item Ausencia de clases innecesarias o irrelevantes.
    \item Tipo correcto de elemento en UML (clase, interfaz, etc.).
\end{itemize}
\subsection{Contenido - 2}
\begin{itemize}
    \item Atributos y métodos principales de cada clase
    \item Modificadores de visibilidad.
    \item Tipos, parámetros y retornos.
    \item Excepciones lanzadas por los métodos.
\end{itemize}

\section{Código - 7 puntos}


\subsection{Main - 1 punto}
\begin{itemize}
    \item Creación y utilización correcta de objetos.
    \item Lectura/carga inicial de ficheros integrada correctamente en el arranque, aunque esté delegada en funciones auxiliares o en otra clase.
    \item Separación del \texttt{main} en funciones auxiliares cuando corresponde, evitando un main monolítico.
    \item Ausencia de lógica de dominio en \texttt{main}.
\end{itemize}

\subsection{Excepciones - 1 punto}
\begin{itemize}
    \item Creación de excepciones propias necesarias.
    \item Lanzamiento de excepciones en el lugar adecuado.
    \item Captura de excepciones en el lugar adecuado.
    \item Capturas que no oculten errores ni vacíen el flujo del programa (\texttt{catch} genérico, \texttt{catch} vacío, sin feedback, etc.).
    \item Gestión de excepciones de entrada/salida.
\end{itemize}

\subsection{Implementación general - 5 puntos}
\subsubsection{Navegación por menús - 0.70}
\begin{itemize}
    \item Clase \texttt{Menu} bien modularizada en métodos auxiliares.
    \item Delegación correcta en las clases del sistema, respetando la encapsulación.
    \item Flujo de uso comprensible para el usuario.
    \item Lógica de navegación sencilla y eficiente, sin duplicar código innecesariamente.
\end{itemize}
\subsubsection{Equipar y desequipar - 1.20}
\begin{itemize}
    \item Movimiento correcto entre inventario y equipo.
    \item Reemplazo de equipo sin perder ni duplicar objetos.
    \item Actualización coherente de ataque y defensa.
    \item Uso de \texttt{IEquipable} y polimorfismo.
    \item Distinción correcta entre arma y armaduras.
    \item Encapsulación entre personaje, inventario y equipo.
\end{itemize}
\subsubsection{Comprar en tienda - 1.00}
\begin{itemize}
    \item Movimiento correcto del objeto desde la tienda al inventario del personaje.
    \item Actualización coherente del dinero del personaje y del precio del objeto.
    \item La tienda deja de ofrecer el objeto comprado.
    \item Conservación de la información completa del objeto comprado, incluido metal.
    \item Encapsulación entre menú, tienda, personaje e inventario.
\end{itemize}
\subsubsection{Guardar personaje - 0.70}
\begin{itemize}
    \item Escritura del estado actual del personaje en fichero.
    \item Conservación de datos principales, equipo e inventario.
    \item Formato coherente con el formato de lectura.
    \item Uso adecuado de escritura de ficheros sin mezclar responsabilidades.
    \item Encapsulación: el guardado no debe requerir manipular desde fuera las estructuras internas del personaje.
\end{itemize}
\subsubsection{Uso de \texttt{toString} - 0.20}
\begin{itemize}
    \item Sobrescritura correcta de \texttt{toString} en las clases donde aporta información.
    \item Salida legible y útil para revisar el estado del sistema.
    \item Utilización de \texttt{toString} de forma adecuada en las clases.
\end{itemize}
\subsubsection{Herencia de \texttt{Objeto} - 1.10}
\begin{itemize}
    \item \texttt{Objeto} como clase base.
    \item Herencia correcta en las clases de objetos (armas, armaduras, etc.).
    \item Atributos y comportamiento común ubicados en la clase base.
    \item Constructores bien encadenados.
    \item Ausencia de duplicación innecesaria entre clases hijas.
\end{itemize}
\subsubsection{Uso de \texttt{enum} - 0.10}
\begin{itemize}
    \item Uso cuando hay conjunto cerrado de valores.
    \item Integración real en la lógica.
\end{itemize}

\section{Defensa}
La defensa forma parte de la evaluación. El alumnado debe poder explicar las decisiones de diseño e implementación tomadas en su práctica de forma clara y razonada. Se valorará la capacidad de argumentación y la comprensión profunda del código y el diseño, así como la capacidad de responder a preguntas sobre posibles mejoras o alternativas.
